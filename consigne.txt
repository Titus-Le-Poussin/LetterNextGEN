--------------------------------------------------------------------------------
2.1 TAILWIND CSS - Framework CSS utilitaire
--------------------------------------------------------------------------------

    QU'EST-CE QUE C'EST ?
    Tailwind est un framework CSS qui fonctionne avec des classes utilitaires.
    Au lieu d'ecrire du CSS personnalise, on utilise des classes predefinies
    directement dans le HTML.

    EXEMPLE SIMPLE :
    Au lieu d'ecrire :
        .mon-bouton { background-color: blue; padding: 10px; border-radius: 5px; }

    On ecrit directement dans le HTML :
        class="bg-blue-500 p-4 rounded"

    AVANTAGES :
        - Developpement plus rapide
        - Pas besoin de nommer des classes CSS
        - Design coherent automatiquement
        - Fichier CSS final optimise (uniquement ce qui est utilise)

    PREREQUIS :
        - Node.js installe sur l'ordinateur
        - Initialisation via npm (gestionnaire de paquets)


--------------------------------------------------------------------------------
2.2 SUPABASE - Backend as a Service
--------------------------------------------------------------------------------

    QU'EST-CE QUE C'EST ?
    Supabase est une plateforme qui fournit tout le backend d'une application
    sans avoir a coder un serveur. C'est une alternative open-source a Firebase.

    CE QUE SUPABASE FOURNIT :

        1. Base de donnees PostgreSQL
           - Stocker les donnees (prompts, templates, historique)
           - Interface visuelle pour gerer les tables
           - Requetes SQL ou API automatique

        2. Authentification
           - Systeme de connexion/inscription pret a l'emploi
           - Gestion des sessions utilisateur
           - Connexion par email, Google, GitHub, etc.

        3. Edge Functions
           - Code execute cote serveur (pas dans le navigateur)
           - Permet de proteger les cles API sensibles
           - Ecrit en TypeScript/JavaScript

        4. Stockage de fichiers
           - Heberger des images, PDF, etc.
           - Gestion des permissions d'acces

    POURQUOI L'UTILISER DANS CE PROJET ?
        - Stocker les prompts personnalises des utilisateurs
        - Stocker les modeles de lettres personnalises
        - Gerer les comptes utilisateurs
        - Proteger la cle API des services IA (jamais exposee au navigateur)
        - Garder un historique des lettres generees

    COUT :
        - Gratuit : 500 MB base de donnees, 1 GB stockage, 500K Edge Functions/mois
        - Largement suffisant pour ce projet

    SITE : https://supabase.com


--------------------------------------------------------------------------------
2.3 APIs D'INTELLIGENCE ARTIFICIELLE
--------------------------------------------------------------------------------

    POURQUOI UTILISER PLUSIEURS APIs ?

    Chaque API a ses avantages et limites. En avoir plusieurs permet :
        - Plus de requetes gratuites pour les tests
        - Une solution de secours si une API est indisponible
        - Comparer la qualite des reponses

    +---------------------------------------------------------------------------+
    |                         COMPARATIF DES APIs                               |
    +-----------------+--------------------------+------------------------------+
    |                 | CLAUDE (Anthropic)       | GEMINI (Google)              |
    +-----------------+--------------------------+------------------------------+
    | Cout            | ~3$/million tokens       | Gratuit (quota genereux)     |
    | Qualite         | Excellente               | Tres bonne                   |
    | Inscription     | Carte bancaire requise   | Compte Google suffit         |
    | Limite gratuite | Credits initiaux limites | 60 requetes/minute           |
    | Ideal pour      | Production finale        | Developpement et tests       |
    +-----------------+--------------------------+------------------------------+

    STRATEGIE RECOMMANDEE :
        1. Pendant le developpement : Utiliser Gemini (gratuit, pas de CB)
        2. Pour les tests finaux : Tester avec Claude
        3. En production : Proposer le choix a l'utilisateur ou utiliser Claude

    APIs SUPPLEMENTAIRES POSSIBLES (toutes gratuites avec limites) :
        - Mistral AI : API francaise, bonne qualite
        - Groq : Tres rapide, modeles open-source
        - OpenRouter : Acces a plusieurs modeles via une seule API

    POURQUOI PASSER PAR SUPABASE (Edge Function) ?

    La cle API des services IA est SECRETE. Si on l'utilise directement
    dans le navigateur, n'importe qui peut la voler en inspectant le code.

    L'Edge Function agit comme intermediaire :
        - La cle API reste sur le serveur Supabase (securisee)
        - Le navigateur envoie seulement l'offre d'emploi
        - Supabase fait l'appel a l'API IA et renvoie le resultat

--------------------------------------------------------------------------------
ETAPE 3.4 : Configurer la securite (Row Level Security)
--------------------------------------------------------------------------------

    QU'EST-CE QUE LE RLS ?
    Row Level Security permet de definir des regles d'acces aux donnees.
    Chaque utilisateur ne peut voir/modifier que SES propres donnees.

    REGLES A CREER POUR CHAQUE TABLE :

        1. SELECT (lecture) :
           - Les prompts/templates par defaut sont visibles par tous
           - Les prompts/templates personnels sont visibles par leur proprietaire

        2. INSERT (creation) :
           - Un utilisateur connecte peut creer ses propres donnees

        3. UPDATE (modification) :
           - Un utilisateur peut modifier uniquement ses propres donnees

        4. DELETE (suppression) :
           - Un utilisateur peut supprimer uniquement ses propres donnees

8. PHASE 5 : INTEGRATION DES APIs IA
================================================================================

OBJECTIF : Permettre l'analyse automatique des offres d'emploi par IA

--------------------------------------------------------------------------------
ETAPE 5.1 : Installer Supabase CLI
--------------------------------------------------------------------------------

    Le CLI Supabase permet de creer et deployer des Edge Functions
    depuis son ordinateur.

    ACTIONS :
        1. Installer le CLI via npm
        2. Se connecter a son compte Supabase
        3. Lier le projet local au projet Supabase distant


--------------------------------------------------------------------------------
ETAPE 5.2 : Creer l'Edge Function "generate-letter"
--------------------------------------------------------------------------------

    ROLE DE CETTE FONCTION :
        - Recevoir l'offre d'emploi et le prompt depuis le navigateur
        - Appeler l'API IA choisie (Claude ou Gemini)
        - Parser la reponse JSON
        - Renvoyer les champs extraits au navigateur

    PARAMETRES D'ENTREE :
        - jobOffer : Le texte de l'offre d'emploi
        - prompt : Le prompt a utiliser pour l'analyse
        - aiProvider : "claude" ou "gemini" (choix de l'API)

    SORTIE :
        - JSON avec les champs : field1 a field7 (voir details ci-dessous)


--------------------------------------------------------------------------------
ETAPE 5.3 : Format de reponse attendu de l'IA
--------------------------------------------------------------------------------

    L'IA doit retourner un JSON avec ces champs :

    +--------+----------------------------------------------------------+
    | Champ  | Description                                              |
    +--------+----------------------------------------------------------+
    | field1 | Domaine de l'entreprise                                  |
    |        | Ex: "la mobilite et l'innovation ferroviaire"            |
    +--------+----------------------------------------------------------+

ETAPE 5.4 : Configurer les cles API dans Supabase
--------------------------------------------------------------------------------

    Les cles API doivent etre stockees en tant que "secrets" dans Supabase.
    Elles ne sont jamais visibles dans le code ni accessibles depuis le navigateur.

    SECRETS A CONFIGURER :
        - ANTHROPIC_API_KEY : Cle pour Claude
        - GEMINI_API_KEY : Cle pour Gemini


--------------------------------------------------------------------------------
ETAPE 5.5 : Gestion des erreurs et fallback
--------------------------------------------------------------------------------

    ERREURS A GERER :

        1. API indisponible :
           - Afficher un message clair a l'utilisateur
           - Proposer de reessayer ou d'utiliser une autre API

        2. Quota depasse :
           - Informer l'utilisateur
           - Basculer automatiquement vers une autre API si possible

        3. Reponse mal formatee :
           - L'IA peut parfois ne pas retourner du JSON valide
           - Nettoyer la reponse (enlever les ```json, texte superflu)
           - Si echec, demander a l'utilisateur de reessayer

        4. Offre d'emploi trop longue :
           - Certaines APIs ont des limites de tokens
           - Tronquer ou avertir l'utilisateur
